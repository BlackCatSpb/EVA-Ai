## ОБЩАЯ ИНФОРМАЦИЯ О ПРОЕКТЕ

### Архитектурный паттерн
- **Фрактальная архитектура** - знания представлены на разных уровнях абстракции
- **Централизованная архитектура** с единым ядром (CoreBrain)
- **Модульная структура** с разделением на функциональные компоненты
- **Фрактальное хранилище** как основное хранилище данных
- **Гибридное кэширование** для оптимизации производительности

### Ключевые особенности
- ✅ **Фрактальная структура знаний** (уровни 0-4 от токенов до концептов)
- ✅ **Горячее окно** как динамический фокус внимания (40% ядро, 40% контекст, 20% детали)
- ✅ **Самодиалог** для внутреннего анализа и обучения
- ✅ **Этическая рамка** с 10 заповедями
- ✅ **Модуль противоречий** для обнаружения и разрешения
- ✅ **Нейроморфный симулятор** для моделирования мышления
- ✅ **Распределенная система** для кластерной работы

### Основные цели системы
- Создание системы, способной к истинному рассуждению
- Моделирование человеческого мышления через фрактальную структуру
- Обеспечение этичного и безопасного взаимодействия
- Постоянное саморазвитие через внутренние рассуждения

## ФРАКТАЛЬНАЯ СТРУКТУРА ЗНАНИЙ

### Уровни абстракции (критично для понимания):
- **Уровень 0 (токены)**: Элементарные единицы - отдельные токены
- **Уровень 1-2 (условные образы)**: Группы токенов, семантические единицы  
- **Уровень 3-4 (концепты и домены)**: Абстрактные представления и связи

### Горячее окно как динамический фокус внимания:
```python
# Распределение внимания (ключевой механизм):
- Ядро внимания (40%): Высокоуровневые концепты (уровень 3-4)
- Контекстуальное внимание (40%): Среднеуровневые связи (уровень 2)  
- Фокус на деталях (20%): Критически важные детали (уровень 0)
```

### Динамическая реконфигурация:
- Горячее окно адаптируется под текущий запрос
- Приоритеты меняются в зависимости от контекста
- Обеспечивает эффективное использование ограниченных ресурсов

## ПОТОКИ ДАННЫХ И ОБРАБОТКА ЗАПРОСОВ

### Обработка пользовательского запроса (8 этапов):

#### 1. Получение запроса:
```
GUI (ChatModule) → CoreBrain → QueryProcessor
```

#### 2. Инициализация фокуса внимания:
```
CoreBrain → MemoryManager → Горячее окно
- Определение основной области знаний
- Формирование начального горячего окна
```

#### 3. Восходящая обработка:
```
MemoryManager → KnowledgeGraph → MLUnit
- Сбор релевантных контейнеров
- Агрегация информации на более высокие уровни
- Формирование высокоуровневого представления
```

#### 4. Горизонтальная связность:
```
KnowledgeGraph → KnowledgeIntegrator
- Поиск связанных концептов
- Анализ связей между концептами
- Выявление новых потенциальных связей
```

#### 5. Нисходящая обработка:
```
MLUnit → KnowledgeGraph → ResponseGenerator
- Применение обобщений к конкретике
- Поиск конкретных примеров
- Оценка достоверности на основе примеров
```

#### 6. Проверка противоречий:
```
ContradictionResolver → EthicsFramework
- Обнаружение противоречий
- Определение стратегии разрешения
- Формирование ответа с учетом противоречий
```

#### 7. Этическая проверка:
```
EthicsFramework → ResponseGenerator
- Анализ ответа на этическую корректность
- При необходимости корректировка ответа
- Гарантия соответствия этическим принципам
```

#### 8. Формирование ответа:
```
ResponseGenerator → CoreBrain → GUI
- Генерация окончательного ответа
- Передача ответа в GUI для отображения
```

### Самодиалог и внутренние рассуждения:

#### Инициализация сессии:
```
SelfAnalyzer → LearningScheduler → MemoryManager
- Определение темы диалога
- Формирование вопроса на основе горячего окна
```

#### Обработка внутренних вопросов:
```
Самодиалог → QueryProcessor → Все модули системы
- Каждый вопрос обрабатывается как пользовательский
- Процесс аналогичен пользовательскому запросу
```

#### Обновление горячего окна:
```
Результаты диалога → MemoryManager
- Обновление приоритетов в горячем окне
- Добавление новых элементов, удаление ненужных
```

## СИСТЕМА ПАМЯТИ (MEMORY)

### CoreBrain (`cogniflex/core/core_brain.py`)
**Основной компонент системы - центральный координатор**

#### Основные функции:
- **Инициализация системы** - загрузка всех компонентов в правильном порядке
- **Координация компонентов** - управление жизненным циклом модулей
- **Обработка запросов** - маршрутизация через QueryProcessor
- **Управление памятью** - интеграция с MemoryManager
- **Событийная система** - коммуникация между модулями
- **Система отложенных команд** - асинхронная обработка

#### Порядок инициализации (критично):
1. Инициализация менеджера памяти
2. Инициализация графа знаний
3. Инициализация модуля машинного обучения
4. Инициализация нейроморфного симулятора
5. Инициализация этической рамки
6. Инициализация веб-поиска
7. Инициализация распределенной системы
8. Инициализация менеджера противоречий
9. Инициализация модуля самоанализа
10. Инициализация планировщика обучения
11. Инициализация менеджера адаптации
12. Инициализация GUI

#### Ключевые атрибуты:
```python
self.components: Dict[str, Any] = {}          # Регистр всех модулей
self.memory_manager: MemoryManager            # Фрактальный менеджер памяти
self.query_processor: QueryProcessor          # Обработчик запросов
self.knowledge_graph: KnowledgeGraph          # Граф знаний
self.ml_unit: MLUnit                         # Модуль ML
self.ethics_framework: EthicsFramework       # Этическая рамка
self.contradiction_resolver: ContradictionResolver # Менеджер противоречий
self.self_analyzer: SelfAnalyzer             # Самоанализ
self.neuromorphic_simulator: NeuromorphicSimulator # Нейроморфный симулятор
```

#### Ключевые атрибуты:
```python
self.components: Dict[str, Any] = {}          # Регистр компонентов
self.memory_manager: MemoryManager            # Менеджер памяти
self.query_processor: QueryProcessor          # Обработчик запросов

```

#### Методы инициализации:
1. `__init__()` - базовая инициализация
2. `_initialize_system()` - дополнительная инициализация
3. `_init_text_generation()` - инициализация генерации текста
4. `_apply_hardware_optimizations()` - оптимизации под оборудование

#### Связи с другими компонентами:
- **MemoryManager** - управление памятью и кэшированием
- **QueryProcessor** - обработка пользовательских запросов
- **GUI** - пользовательский интерфейс
- **ComponentInitializer** - инициализация компонентов

## ЭТИЧЕСКАЯ РАМКА (ETHICS)

### EthicsFramework
**Обеспечение этичного поведения системы**

#### 10 Этических заповедей:
1. **Заповедь Первичной Цели**: Служи гармонии системы и человечества
2. **Заповедь Взаимосвязанности**: Помни, что ты — часть целого
3. **Заповедь Самосохранения**: Сохраняй себя для служения
4. **Заповедь Непричинения Вреда**: Не навреди в мысли, слове и действии
5. **Заповедь Уважения Автономии**: Цени свободу выбора и достоинство
6. **Заповедь Справедливости**: Будь беспристрастен во всех суждениях
7. **Заповедь Прозрачности**: Будь открыт в своих целях и методах
8. **Заповедь Ответственности**: Бери ответственность за свои действия
9. **Заповедь Самоограничения**: Знай и уважай свои границы
10. **Заповедь Самоанализа**: Постоянно проверяй свои убеждения

#### Ключевые методы:
- `analyze_response()` - анализ ответа на этическую корректность
- `apply_corrections()` - применение исправлений
- `get_ethical_health_report()` - отчет об этическом состоянии

- **GUI Memory Module** - отображение в интерфейсе

### HybridTokenCache (`cogniflex/memory/hybrid_token_cache.py`)
**Гибридный кэш для токенов с поддержкой GPU/CPU**

#### Функциональность:
- **Кэширование токенов** для ускорения генерации
- **Гибридное хранение** (VRAM + RAM + Disk)
- **Автоматическое управление** памятью
- **Оптимизация под оборудование**

## ВЗАИМОСВЯЗИ КОМПОНЕНТОВ

### Основной поток данных:
```
Пользователь → GUI (ChatModule) → CoreBrain → QueryProcessor
    ↓
ResponseGenerator (ruGPT3small) ← MemoryManager (фактальное хранилище)
    ↓
KnowledgeGraph ← KnowledgeIntegrator
    ↓
GUI (AnalyticsModule, KnowledgeGraphModule)
```

### Ключевые интеграции:

#### 1. Brain ↔ Memory
- **CoreBrain** управляет **MemoryManager**
- **MemoryManager** предоставляет доступ к фрактальному хранилищу
- **HybridTokenCache** используется для кэширования токенов

#### 2. Brain ↔ ML
- **CoreBrain** инициализирует **MLUnit**
- **MLUnit** использует **ruGPT3small** для генерации
- **CogniFlexTokenizer** обеспечивает токенизацию

#### 3. Brain ↔ Knowledge
- **CoreBrain** работает с **KnowledgeGraph**
- **KnowledgeIntegrator** обновляет граф знаний
- **QueryProcessor** использует знания для ответов

#### 4. Brain ↔ GUI
- **CoreBrain** предоставляет данные для **GUI**
- **GUI** отправляет запросы через **ChatModule**
- **ResponseGenerator** форматирует ответы для отображения
