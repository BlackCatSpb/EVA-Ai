# Архитектура GUI CogniFlex

Документ описывает архитектуру графического интерфейса пользователя системы CogniFlex. GUI-подсистема построена на основе модульной архитектуры с поддержкой событийной шины и интеграции с ядром системы через централизованный интегратор.

## Общая структура

Графический интерфейс CogniFlex состоит из нескольких ключевых компонентов, каждый из которых отвечает за определенную функциональность. Основной точкой входа является модуль `core_gui.py`, который содержит класс `CogniFlexGUI`. Этот класс обеспечивает полнофункциональный интерфейс с поддержкой всех расширенных возможностей системы.

Структура GUI-подсистемы организована следующим образом: в корневом каталоге модуля располагаются основные файлы реализации интерфейса, вспомогательные утилиты и настройки. Модульная архитектура позволяет легко расширять функциональность путем добавления новых модулей без изменения основного кода ядра.

### Основные компоненты

Первым и наиболее важным компонентом является класс `CogniFlexGUI`, расположенный в файле `core_gui.py`. Этот класс инициализирует главное окно приложения, настраивает темы оформления, управляет навигацией между модулями и обеспечивает взаимодействие с системным ядром.

Вторым ключевым компонентом является класс `IntegratedCogniFlexGUI` из файла `integrated_gui.py`. Этот класс представляет альтернативную реализацию интерфейса с поддержкой фрактальной архитектуры.

Третьим основным компонентом выступает модуль чата `ChatModule` в файле `chat_module.py`. Данный модуль реализует полнофункциональный интерфейс для взаимодействия пользователя с системой через текстовые сообщения.

## Модули и их назначение

Система GUI включает восемь основных модулей: чат, аналитика, граф знаний, противоречия, память, обучение, нейроморфика и настройки.

### Модуль чата (ChatModule)

Модуль чата обеспечивает отправку запросов и получение ответов от когнитивного ядра. Поддерживается форматирование Markdown, гиперссылки, многострочный ввод.

### Модуль аналитики (AnalyticsModule)

Модуль аналитики отвечает за визуализацию статистических данных о работе системы.

### Модуль графа знаний (KnowledgeGraphModule)

Модуль графа знаний обеспечивает визуализацию и управление графом знаний системы.

### Модуль противоречий (ContradictionModule)

Модуль противоречий отвечает за обнаружение и разрешение логических противоречий.

### Модуль памяти (MemoryModule)

Модуль памяти отображает информацию о состоянии гибридной системы кэширования.

### Модуль обучения (LearningModule)

Модуль обучения управляет процессом обучения модели на пользовательских данных.

### Модуль нейроморфики (NeuromorphicModule)

Модуль нейроморфики визуализирует работу нейроморфных компонентов системы.

### Модуль настроек (SettingsModule)

Модуль настроек предоставляет интерфейс для конфигурации параметров системы.

## Потоки данных

Взаимодействие между компонентами GUI и ядром системы осуществляется через событийную шину интегратора и очередь задач GUI.

### Обработка запросов пользователя

Запрос пользователя проходит через очередь обработки в фоновом потоке и направляется в ядро системы через интегратор.

### Событийная шина

Событийная шина обеспечивает асинхронный обмен сообщениями между компонентами системы.

## Тематическое оформление

Система поддерживает две темы оформления: светлую и темную.

## Кэширование и сохранение состояния

Система автоматически сохраняет состояние интерфейса и историю чата.
