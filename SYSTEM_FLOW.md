## 🎯 Обзор Системы

ЕВА (EVA) — это когнитивная система искусственного интеллекта с двумя архитектурами:
- **GPU архитектура**: для обработки на графическом процессоре
- **CPU архитектура**: фрактальное хранение и генерация промтов для модулей противоречий, этики и веб-поиска

### Ключевые Инновации
- **GGUF/LlamaCpp**: Быстрый CPU inference (~48 токенов/сек)
- **Иерархия обработки**: Интеграция модулей противоречий, этики, веб-поиска
- **Семантический граф памяти**: Хранение знаний с логическими связями
- **Куратор графа**: Автоматическое создание семантических связей
- **Самодиалог**: Анализ неизвестных понятий через веб-поиск
- **Дообучение GGUF**: Точечное дополнение весов через LoRA

## 📦 Модули Системы

### 1. Ядро (Core)
**Файл**: `eva/core/core_brain.py`

#### Основные Методы:
```python
def process_query(query, user_context, context, max_new_tokens, temperature, top_p, repetition_penalty)
```
**Принцип работы**:
1. Проверяет `qwen_only_mode` в конфигурации
2. Извлекает контекст из графа памяти
3. Формирует промпт с контекстом
4. Вызывает GGUF для генерации
5. Запускает модули (противоречия, этика, веб-поиск)
6. Перегенерирует при необходимости
7. Сохраняет диалог в граф памяти

---

### 2. GGUF/LLamaCpp (Быстрый Inference)
**Файлы**: 
- `eva/mlearning/fractal_model_manager.py`
- `eva/mlearning/hot_deployment/llama_cpp_hot.py`

#### Конфигурация:
```json
{
  "model": {
    "use_llama_cpp": true,
    "gguf_model_path": "eva/models/qwen2.5-0.5b-instruct-q4_0.gguf",
    "llama_cpp_threads": 8,
    "llama_cpp_n_ctx": 4096
  }
}
```

#### Принципы работы:
- **Q4_0 квантование**: 4-битное сжатие для экономии памяти
- **8 потоков**: Параллельная обработка на CPU
- **4096 токенов контекста**: Достаточно для длинных диалогов
- **Автоматическая загрузка**: Модель загружается при старте системы

---

### 3. Модуль Противоречий (Contradiction)
**Файл**: `eva/contradiction/contradiction_manager.py`

#### Методы:
```python
def check_with_context(query, response) -> Dict
```
**Принципы работы**:
- Сравнивает ответ с базой знаний
- Обнаруживает логические противоречия
- Возвращает `significant_count` — количество найденных противоречий

---

### 4. Модуль Этики (Ethics)
**Файл**: `eva/ethics/ethics_framework.py`

#### Методы:
```python
def check_with_context(query, response) -> Dict
```
**Принципы работы**:
- Проверяет ответ на соответствие этическим нормам
- Оценивает `is_ethical` (True/False)
- При нарушениях → перегенерация ответа

### 5. Веб-поиск (Web Search)
**Файлы**:
- `eva/websearch/web_search_engine.py`
- `eva/websearch/search_models.py`

#### Методы:

#### Принципы работы:
- **Пропускает простые фразы**: «Привет», «Как дела», «Спасибо»...

### 8. Самодиалог (Self-Dialog Learning)
**Файлы**:
- `eva/learning/self_dialog_learning.py`
- `eva/learning/self_dialog_new_methods.py`

#### Методы:
```python
def extract_key_concepts(query) -> List[str]
def analyze_unknown_concepts(query, response) -> List[Dict]

```

#### Принципы работы:
1. **Извлечение понятий**: Разбивает запрос на слова и словосочетания
2. **Анализ неизвестного**: Проверяет ответ на «я не знаю»
3. **Веб-поиск по понятиям**: Для каждого неизвестного понятия → веб-поиск
4. **Сохранение в граф**: Результаты сохраняются в knowledge_graph

### 10. Система Дообучения GGUF (Training System)
**Файлы**:
- `eva/training/gguf_training_system.py`
- `eva/training/__init__.py`

#### Архитектура:
```
┌─────────────────────────────────────────────────────────────────┐
│                    Граф Памяти (Верифицированные Узлы)           │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│              GGUFTrainingSystem                                  │
│  1. Извлечение верифицированных знаний                          │
│  2. Подготовка данных (пары вопрос-ответ)                       │
│  3. Обучение ОТДЕЛЬНОГО экземпляра                              │
│  4. Верификация качества                                        │
│  5. Сохранение LoRA адаптеров                                   │
└─────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│          LoRA Adapter (Точечное дополнение весов)                │
│  • Сохраняется отдельно                                          │
│  • Не меняет базовую модель                                      │
│  • Можно откатить                                                │
└─────────────────────────────────────────────────────────────────┘
```

#### Метрики Верификации:
```json
{
  "loss": 0.0,
  "accuracy": 0.0,
  "knowledge_volume": 0,
  "verification_score": 0.0,
  "integrity_check": true/false,
  "nodes_processed": 0,
  "links_learned": 0,
  "training_time": 0.0,
  "generation_quality": 0.0
}
```

#### Ключевые Особенности:
1. **Разделение на две модели**:
   - **Production GGUF**: используется для ответов (НЕ обучается)
   - **Training GGUF**: отдельный экземпляр только для обучения

2. **LoRA адаптеры**:
   - Точечное дополнение весов (не полная перетренировка)
   - Сохраняются в `eva/models/lora_adapters/`
   - Можно откатить, если качество ухудшится

3. **Верификация**:
   - Проверка целостности модели
   - Проверка качества генерации
   - Проверка объема знаний

4. **Запуск (по запросу)**:
   ```python
   brain.gguf_training.start()  # Запуск дообучения
   brain.gguf_training.stop()   # Остановка
   status = brain.gguf_training.get_status()  # Статус
   ```

## 🔄 Семантическая Логика Связей

### Принцип Формирования Связей

Граф памяти формируется на основе **семантических ассоциаций**, а не просто «существительное — определение».

**Пример цепочки для слова «снег»**:
```
снег
  ├── белый (цвет)
  ├── холодный (температура)
  ├── зимний (сезон)
  ├── искрящийся (качество)
  ├── пушистый (текстура)
  ├── первый (время)
  ├── сугроб (объект)
  └── февраль (месяц)
```

**Принципы связи**:
1. **Атрибутивные**: снег — белый, холодный
2. **Контекстные**: снег — зима, февраль
3. **Объектные**: снег — сугроб, лыжи
4. **Эмоциональные**: снег — красивый, радостный
5. **Сенсорные**: снег — искрящийся, пушистый, холодный
