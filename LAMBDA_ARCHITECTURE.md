# Архитектура λ_d: Математическое обоснование и методы

## 1. Математическая основа

### 1.1 Рекурренция λ_d

Ключевая идея λ_d — замена фиксированного линейного преобразования (как в SSM/Mamba) на **контент-зависимый линейный оператор** A(h), спектр которого вычисляется по текущему скрытому состоянию:

```
h_{t+1} = A(h_t) · h_t
```

где `A(h) = V · diag(α(h) · λ) · V⁻¹`.

### 1.2 Корни Фибоначчи (спектр λ_k)

Спектральные радиусы λ_k получены как наибольшие действительные корни характеристического полинома d-шаговой матрицы Фибоначчи:

```
x^k = x^{k-1} + x^{k-2} + ... + x + 1
```

Это эквивалентно `x^(k+1) - 2·x^k + 1 = 0` при x ≠ 1.

**Вывод:** для k=2 получаем `x² = x + 1`, корень φ = (1+√5)/2 ≈ 1.618. Для k=3: `x³ = x² + x + 1`, корень ≈ 1.839. Предел при k→∞: 2.

**Почему Фибоначчи, а не произвольный спектр?** Последовательность Фибоначчи — простейшая рекурренция с экспоненциальным ростом. Её собственные числа λ_k образуют естественный алфавит скоростей роста информации. Каждый λ_k соответствует горизонту памяти: при λ = 1.618 информация затухает как φ^(-t), при λ → 2.0 — почти не затухает.

**Вычислительный метод (Newton):**
```
λ = λ - (λ^k - λ^(k-1) - λ^(k-2) - ... - 1) / (k·λ^(k-1) - (k-1)·λ^(k-2) - ... - 1)
```
Сходится за 5-6 итераций от λ₀ = 1.9.

### 1.3 Контент-зависимое управление (gating)

Для каждого токена на каждом слое вычисляется α(h) ∈ Δ^K — выпуклая комбинация K корней:

```
α(h) = softmax(4.0 · (W_gate · rms_norm(h) + b_gate))
λ_eff = α · λ     ∈ (λ₂, λ_K) ⊂ (1.618, 2.0)
```

**Ключевое свойство:** матрица A(h) меняется от токена к токену. Токены с высокой информационной ценностью (редкие слова, начала предложений) могут использовать λ_eff → 2.0 (долгая память), а частые/стоп-слова — λ_eff → 1.618 (короткая память).

**Множитель scale=4.0:** без него softmax при D=256 даёт почти равномерное распределение (энтропия 1.39/1.39). При scale=4.0 энтропия падает до 1.26/1.39 — гейты дифференцируются.

### 1.4 Ортогональный собственный базис V

V ∈ O(D) — случайная ортогональная матрица, построенная как произведение D случайных отражений Хаусхолдера:

```
V = H_1 · H_2 · ... · H_{n_refl}
H_i = I - 2 · v_i · v_i^T / ||v_i||²    где v_i ~ N(0, I_D)
```

- V фиксируется после инициализации (buffer, не trainable)
- V⁻¹ = V^T (ортогональность) — обращение за O(D²) не требуется
- Каждый слой имеет свой V_l с разным seed

**Зачем?** Без V все моды λ действуют независимо по осям (покомпонентное масштабирование). V смешивает оси, распределяя спектральное сжатие/растяжение по всем D измерениям.

### 1.5 Zeckendorf как нелинейность

Чисто линейная модель A(h)·h редуцируется к покомпонентному масштабированию. Нелинейность в λ_d вносится **дискретным ограничением Цекендорфа**:

**Теорема Цекендорфа:** каждое натуральное число имеет единственное представление в виде суммы НЕСОСЕДНИХ чисел Фибоначчи.

Это накладывает жёсткое ограничение: если на уровне k выбран бит 1, то на уровне k-1 бит **обязательно 0**. Это предотвращает коллапс к одной моде и создаёт древовидную структуру пространства токенов.

## 2. Компоненты архитектуры

### 2.1 Общая схема

```
tokens → Embed(D) → h₀
for l in 1..L:
    h = h + LDBlock_l(rms_norm(h))     # λ_d преобразование
    h = h + MLP_l(rms_norm(h))          # SwiGLU + LoRA
h = rms_norm(h)
logits = h @ lm_head^T                  # или ZeckendorfReadout
P(token) = softmax(logits)
```

### 2.2 LDBlock (ядро)

```
→ h_norm = rms_norm(h)
→ λ_eff = softmax(4.0 · (h_norm@W_gate + b)) @ λ_k
→ Δ = V · diag(λ_eff) · V^T · h_norm
→ Δ = Δ / max(λ_eff + eps)
→ h_out = h + Δ          (или просто Δ, если residual=False)
```

**rms_norm:** `x / sqrt(mean(x²) + eps) * w`. Гарантирует RMS=1 после нормы, что ограничивает ||Δ|| ≤ √D независимо от ||h||.

**Spectral clamping:** деление на max(λ_eff) гарантирует, что наибольшее собственное значение residual-оператора I + V·Λ̂·V⁻¹ не превышает 2. Это даёт линейный рост нормы по глубине (не экспоненциальный).

**residual=False:** используется при внешнем residual (Qwen2-стиль: `h = h + LDBlock(rms_norm(h))`). Внутренний residual привёл бы к удвоению прироста на слой.

### 2.3 LDMLP (SwiGLU + LoRA)

```
z = silu(h @ W_gate^T) * (h @ W_up^T)
h_out = z @ W_down^T
```

Используются веса из Qwen2.5-0.5B (заморожены). LoRA-адаптеры:
```
W_eff = W_buffer + A @ B
```
A ∈ ℝ^{D×r}, B ∈ ℝ^{r×I} для gate/up; A ∈ ℝ^{I×r}, B ∈ ℝ^{r×D} для down.

**Зачем LoRA, а не полное обучение?** MLP — 96.5% параметров. Даже LoRA rank=256 — это всего 0.7M доп. параметров на слой против 13.1M полных.

### 2.4 ZeckendorfReadout (инференс)

**Постановка:** lm_head = W_embed^T ∈ ℝ^{V×D} — огромная матрица (896×50000 = 44.8M параметров). Замена на древовидный декодер с K·4·D = 14.3K параметров.

**Алгоритм декодирования (жадный):**

```
bit_string = []
state = 0
for k = K-1 .. 0:
    if state == 1:
        bit = 0               # ограничение Цекендорфа
    else:
        logit_0 = h · c[k, 0, 0]
        logit_1 = h · c[k, 0, 1]
        bit = argmax(softmax([logit_0, logit_1]))
    bit_string.append(bit)
    state = bit
token_id = sum(bit_string[k] * F[k])
```

**Центроиды** c[k, prev_bit, digit] ∈ ℝ^D обучаются через стандартный CE loss: каждый центроид учится представлять регион скрытого пространства, соответствующий данному биту на данном уровне. Градиенты проходят через argmax через gumbel-softmax (при обучении).

**Почему h·c (линейные логиты), а не φ^(h·c)?** Изначально использовалось φ^(h·c), но φ^x ≡ exp(x·ln φ) — это softmax с температурой T = 1/ln φ ≈ 2.08. Никакого математического преимущества, только риск насыщения экспоненты. Заменено на h·c.

### 2.5 LoRALinear

Адаптер для любой frozen-матрицы W:
```
forward(x): x @ W^T + (x @ B^T) @ A^T      # или x @ (W + A@B)^T
```

## 3. Анализ стабильности

### 3.1 Спектральный радиус residual

Оператор: h_out = h + V·Λ̂·V⁻¹·norm(h)

Собственные числа: 1 + λ_eff_i / max(λ_eff) ∈ (1, 2].

**Это НЕ RNN.** LDBlock применяется 1 раз на слой, параллельно по всем позициям. Если бы он итерировался во времени как x_{t+1} = A·x_t, спектральный радиус > 1 вызвал бы взрыв. Но residual-архитектура с pre-norm — это:

```
h_L = h_0 + Σ_{l=0}^{L-1} V_l · Λ̂_l · V_l^T · rms_norm(h_l)
```

Каждое слагаемое суммы ограничено по норме: ||V_l · Λ̂_l · V_l^T · rms_norm(h_l)|| ≤ √D (так как rms_norm даёт ||h_norm|| = √D, а V ортогонален). ОТСЮДА:

```
||h_L|| ≤ ||h_0|| + L · √D
```

**Линейный рост по глубине** — идентично трансформеру с pre-norm.

### 3.2 NaN в fp16

**Проблема:** любая операция `.half()` в autograd-графе на CUDA (MX550) даёт 100% NaN. Причина: смешанная точность (fp16 gradients + fp32 weights) создаёт градиенты, непредставимые в fp16.

**Решение:** все операции в чистом fp32. Никаких `.half()`, `.float()`, `.to(dtype)`. Memory penalty есть, но стабильность абсолютная.

### 3.3 rms_norm underflow

При малых значениях `rms_norm` может дать eps-защиту: `x / sqrt(mean(x²) + eps) * w`. Без eps при очень малых x (после norm) возможно деление на ~0. Добавлен `clamp(min=eps)` в знаменатель.

## 5. Нелинейности в λ_d

В модели присутствуют следующие нелинейности:

1. **Softmax в гейтах:** α = softmax(4.0·(W_gate·h + b)) — превращает линейную проекцию в распределение
2. **SwiGLU (MLP):** silu(x)·y — основная нелинейность трансформера
3. **Zeckendorf constraint:** запрет соседних 1 — дискретная нелинейность дерева
4. **RMS Norm:** деление на sqrt(E[x²]) — адаптивная нелинейность масштабирования

Линейная часть модели (V·Λ·V⁻¹·h) не может одна дать достаточную выразительность — нелинейности критичны.

## 6. Сравнение с альтернативами

| Аспект | Transformer | Mamba | λ_d |
|--------|-------------|-------|-----|
| Внимание | QK^T — O(L²) | O(L) сканирование | O(L) (нет внимания) |
| Зависимость от позиции | RoPE/AliBi | 1D conv | Нет (residual feed-forward) |
| Параметры внимания | 4·D² | D·Δ (selective scan) | D·K (гейты) |
| Спектр | Фиксирован | Фикс. (HIPPO) | α(h)-зависимый |
| Длина контекста | 8K-128K | 128K+ | Не ограничен (нет внимания) |
| Нелинейность | Softmax/SwiGLU | SiLU/SwiGLU | Zeckendorf + SwiGLU |

**Ключевое отличие λ_d:** каждый токен имеет свой спектральный радиус. Это аналог «selective scan» из Mamba, но реализованный через гейты над корнями Фибоначчи, а не через входозависимый Δ.
