# Архитектура λ_d: Математическое обоснование и методы

## 1. Математическая основа

### 1.1 Рекурренция λ_d

Ключевая идея λ_d — замена фиксированного линейного преобразования (как в SSM/Mamba) на **контент-зависимый линейный оператор** A(h).

**Тренировка (residual, параллельно по всем позициям):**
```
h_out = h + V · diag(α(h)·λ) · V⁻¹ · norm(h + conv(h))
```

**Инференс (авторегрессивно, один токен за раз):**
```
h_t = h_{t-1} + V · diag(α(h_t)·λ) · V⁻¹ · norm(h_{t-1} + conv([h_{t-1}, ..., h_{t-4}]))
```

### 1.2 Корни Фибоначчи (спектр λ_k)

Спектральные радиусы λ_k получены как наибольшие действительные корни характеристического полинома d-шаговой матрицы Фибоначчи:

```
x^k = x^{k-1} + x^{k-2} + ... + x + 1
```

**Вывод:** для k=2 получаем `x² = x + 1`, корень φ = (1+√5)/2 ≈ 1.618. Для k=3: `x³ = x² + x + 1`, корень ≈ 1.839. Предел при k→∞: 2.

**Почему Фибоначчи?** Собственные числа λ_k образуют естественный алфавит скоростей роста информации. Каждый λ_k соответствует горизонту памяти: при λ = 1.618 информация затухает как φ^(-t), при λ → 2.0 — почти не затухает.

**Вычислительный метод (Newton):** сходится за 5-6 итераций от λ₀ = 1.9.

### 1.3 Контент-зависимое управление (gating) + блоковый спектр

Для каждого токена на каждом слое α(h) ∈ Δ^K распределяет вес между K корнями:

```
α(h) = softmax(4.0 · (W_gate · rms_norm(h) + b_gate))  # (B, L, K)
```

λ_eff — **вектор размера D**, где каждая из K групп размерности D//K получает λ_k · α_k:

```
λ_eff = (α ⊙ λ).repeat_interleave(D//K, dim=-1)  # (B, L, D)
```

Тогда полный оператор:

```
h_proj = h_norm @ V^T          # в eigenbasis
h_scaled = h_proj * λ_eff       # поблочное масштабирование
Δ = h_scaled @ V                # обратно в исходный базис
h_out = h + Δ
```

Без скалярного clamping — каждая eigen-мода масштабируется независимо. Норма Δ ограничена: ||Δ|| ≤ max(λ)·√D ≈ 2·√D.

**Ключевое свойство:** матрица A(h) меняется от токена к токену поблочно — разные eigen-направления получают разное усиление.

**Множитель scale=4.0:** без него softmax даёт почти равномерное распределение. При scale=4.0 гейты дифференцируются.

**Важно: скалярный λ_eff + clamping был багом.** Старая реализация давала identity. Исправлено на блоковый вектор λ_eff без clamping.

### 1.4 Ортогональный собственный базис V

V ∈ O(D) — случайная ортогональная матрица, построенная как произведение 32 отражений Хаусхолдера. V фиксируется после инициализации (buffer, не trainable). V⁻¹ = V^T. Каждый слой имеет свой V_l.

**Зачем?** Без V все моды λ действуют независимо по осям. V смешивает оси, распределяя спектральное сжатие/растяжение по всем D измерениям.

### 1.5 Zeckendorf как нелинейность

Чисто линейная модель A(h)·h редуцируется к покомпонентному масштабированию. Нелинейность в λ_d вносится **дискретным ограничением Цекендорфа**: каждое натуральное число имеет единственное представление в виде суммы НЕСОСЕДНИХ чисел Фибоначчи. Если на уровне k выбран бит 1, то на уровне k-1 бит **обязательно 0**.

## 2. Компоненты архитектуры

### 2.1 Общая схема

```
tokens → Embed(D) → h₀
for l in 1..L:
    h_conv = causal_conv1d(h)              # локальный контекст (n-граммы)
    h = h + LDBlock(rms_norm(h + h_conv))  # λ_d спектральное преобразование
    h = h + BottleneckMLP(rms_norm(h))     # D → bottleneck → D
h = rms_norm(h)
logits = h @ lm_head^T                     # или ZeckendorfReadout
P(token) = softmax(logits)
```

### 2.2 CausalConv1d (кросс-токен микс)

Depthwise causal 1D-свёртка с kernel_size=4, padding=3 слева (causal):

```
h_conv = Conv1d(h, kernel=4, padding=3_left, groups=D)
h_norm = rms_norm(h + h_conv)
```

- Веса заморожены после инициализации (randn*0.1)
- Применяется ДО pre-norm, даёт локальный контекст 4 токена
- Решает проблему позиционной слепоты: без неё модель — bag-of-words

### 2.3 LDBlock (ядро)

```
→ h_norm = rms_norm(h + conv(h))
→ α = softmax(4.0 · (h_norm@W_gate + b))
→ λ_eff = (λ_k · α).repeat_interleave(D//K, dim=-1)
→ h_proj = h_norm @ V^T
→ h_scaled = h_proj * λ_eff
→ Δ = h_scaled @ V
→ h_out = h + Δ
```

**Нет clamping.** Норма Δ ограничена max(λ)·√D ≈ 2·√D.

**Почему K блоков, а не D гейтов?** W_gate — D×K (896×4 = 3.6K параметров). Если бы каждый из D каналов имел свой гейт, было бы D² (800K).

### 2.4 BottleneckMLP (единственный MLP)

**Примечание:** Разделы «LDMLP (SwiGLU + LoRA)» и «LoRALinear» удалены. В текущей архитектуре MLP — плотный Bottleneck без LoRA.

```
z = SiLU(W_up @ h)
h_out = W_down @ z
```

- W_up: D → bottleneck (256), W_down: bottleneck → D
- bias=False, SiLU активация
- **Полный ранг 256** против rank=16 у старого LoRA
- Все веса trainable (5.5M на 12 слоёв)

**Почему bottleneck, а не LoRA?** LoRA rank=16 на D=896, I=4864 давал эффективный ранг 16. Bottleneck 256 даёт ранг 256 при сопоставимом числе параметров.

### 2.5 ZeckendorfReadout (альтернативный выход)

**Идея:** lm_head = W_embed^T ∈ ℝ^{V×D} — огромная матрица. Замена на древовидный декодер с K·4·D параметров.

**Алгоритм декодирования (жадный):**

```
bit_string = []
state = 0
for k = K-1 .. 0:
    if state == 1: bit = 0
    else: bit = argmax(softmax([h·c[k,0,0], h·c[k,0,1]]))
    bit_string.append(bit)
    state = bit
token_id = sum(bit_string[k] · F[k])
```

**Центроиды** c[k, prev_bit, digit] ∈ ℝ^D обучаются через CE loss.

## 3. Анализ стабильности

### 3.1 Спектральный радиус residual

Оператор: h_out = h + V·Λ̂·V⁻¹·norm(h + conv(h))

Поскольку α ∈ Δ^K (симплекс, Σα_k = 1) и λ_k < 2, максимальное значение λ_eff,i = λ_k · α_k ≤ λ_max · 1 < 2. Следовательно, собственные числа residual-оператора: 1 + λ_eff,i ∈ (1, 3). На практике спектральный радиус ≈ 1 + λ_max ≤ 2 (ограничение через RMSNorm и SiLU-сжатие).

**Это НЕ RNN.** LDBlock применяется 1 раз на слой, параллельно по всем позициям. Residual-архитектура с pre-norm даёт:

```
h_L = h_0 + Σ_{l=0}^{L-1} V_l · Λ̂_l · V_l^T · rms_norm(h_l + conv(h_l))
```

Каждое слагаемое ≤ √D. Отсюда ||h_L|| ≤ ||h_0|| + L·√D — **линейный рост по глубине**, идентично трансформеру с pre-norm.

### 3.2 NaN в fp16

Любая операция `.half()` в autograd-графе на MX550 даёт 100% NaN. Решение: все операции в чистом fp32.

### 3.3 rms_norm underflow

Добавлен `clamp(min=eps)` в знаменатель для защиты от деления на ~0 при малых x.

## 6.5 Subspace Multiplexing — мультиагентность на одной сети

**Ключевое архитектурное свойство λ_d, отсутствующее у трансформеров.**

Ортогональный базис V ∈ O(D) можно разделить на N независимых ортогональных подпространств:

```
V = [V₁ | V₂ | ... | Vₙ]   где V_i ∈ ℝ^{D × (D/N)},  V_i^T · V_j = 0 при i≠j
```

Каждый агент работает в своём подпространстве, но использует **общую сенсорную кору** (Embedding + Causal Conv1d) и **общий BottleneckMLP**. Разделение происходит через спектральные гейты α(h):

- **Нижние слои** (ближе к conv): λ → φ (короткая память, синтаксис) — агенты обрабатывают общий входящий текст
- **Верхние слои** (ближе к выходу): λ → 2.0 (долгая память) — каждый агент накапливает свою историю в своём подпространстве

**Преимущества:**
- Одна сеть весом 95M = 4+ агента. GPT потребовала бы 4 × 95M = 380M.
- Общая сенсорная кора — агенты видят мир одинаково, но интерпретируют по-разному
- Никакого оверхеда на коммуникацию: агенту достаточно прочитать подпространство соседа (скалярное произведение в ℝ^D)

## 8. Инференс: RNN-режим

На тренировке LDBlock применяется параллельно (как FFN в трансформере). На инференсе — **авторегрессивно**, по одному токену за раз, как RNN:

```
h_t = h_{t-1} + V · Λ̂(h_t) · V^T · norm(h_{t-1} + conv([h_{t-1}, h_{t-2}, h_{t-3}, h_{t-4}]))
```

Causal Conv1d хранит буфер на 3 предыдущих токена (4.5K параметров). Всё остальное — скалярные умножения и матричные произведения на одном векторе. **Никакого роста стоимости с длиной контекста.**

## 10. Эмпирический спектральный анализ (Phase 2, step 25000)

### 10.1 V-энергия: не концентрация, а ротация

Каждый слой имеет свой V_l ∈ O(D) — случайную ортогональную матрицу. Анализ проекции скрытых состояний на V-базис каждого слоя показал, что **V_l остаётся равномерной на всех слоях**:

```
Layer  0: 50% энергии в 352 колонках, 90% в 744
Layer  5: 50% энергии в 351 колонках, 90% в 752
Layer 11: 50% энергии в 346 колонках, 90% в 755
```

Никакой концентрации — распределение энергии близко к равномерному (против 448 для 50%, 806 для 90% при полной равномерности). V_i не «учится» — это чистая ротация, **случайная ортогональная матрица, меняющая оси координат**.

**Откуда анизотропия (×1.3e9), обнаруженная в gate analysis?** Она возникает только при проекции скрытых состояний ВСЕХ слоёв через V_12 (последнего слоя). Разные слои накапливают информацию в разных направлениях, и последний V_12 «видит» эту вариацию. Каждый отдельный V_l просто поворачивает пространство, но накопленная через слои информация создаёт неоднородность в результирующем базисе.

**Вывод:** V — это не механизм концентрации, а механизм **перемешивания осей**. Без V все K мод λ действовали бы независимо по ортогональным направлениям (эффективно — модель с K=4 измерениями). V «размазывает» спектральное управление по всем D=896 измерениям, давая гейтам богатое пространство для манёвра.

### 10.2 Gate-профили: лингвистическая специализация

Гейты α(h) ∈ Δ^4 для разных токенов имеют **устойчивые, различные профили**:

| Токен | m1 (λ=1.618) | m2 (λ=1.839) | m3 (λ=1.928) | m4 (λ=1.966) | Интерпретация |
|-------|:---:|:---:|:---:|:---:|------|
| `,` | .215 | .164 | .264 | .357 | Баланс, m4 доминирует |
| `.` | .223 | **.277** | .162 | .337 | m2 повышен — конец предложения |
| `—` | .120 | .209 | **.400** | .272 | m3 доминирует — пауза/пояснение |
| `и` | **.148** | .202 | .200 | **.451** | m1 минимален — сочинительная связь |
| `в` | .229 | .173 | **.156** | **.443** | m3 подавлен, m4 усилен |
| `на` | .265 | .276 | **.140** | .320 | m3 минимален |
| `не` | .262 | .171 | **.084** | **.484** | m3 экстремально подавлен |
| `но` | .338 | .142 | .132 | .388 | m2 подавлен — противительная связь |
| `ный` | .301 | **.108** | .166 | .424 | m2 минимален — адъективный суффикс |
| `+` | **.373** | .245 | **.048** | .334 | m3 почти ноль — символ |

**Ключевые наблюдения:**
1. **Mode 4 (λ→2.0)** — всегда доминирующий (0.27-0.48). Это «глобальный контекст» — информация, которая не затухает.
2. **Mode 3 (λ=1.928)** — самый изменчивый (.05-.40). Высок для `—` (тире, пауза), низок для `не`, `но` (отрицание/противительность).
3. **Mode 1 (λ=1.618)** — высок для `+`, `но`; низок для `и` (сочинение). Возможно, кодирует локальную синтаксическую структуру.
4. Mode 2 — поднимается на `.` (конец предложения), падает на `ный` (суффикс внутри слова).

**Интерпретация:** λ_d обучается частотно-избирательному вниманию — разные λ_k соответствуют разным временным масштабам обработки текста. Гейты выбирают, какой масштаб важен для данного токена. Знаки препинания, союзы, отрицания, суффиксы — все имеют характерные «спектральные подписи».

### 10.3 State interpolation

Линейная интерполяция между скрытыми состояниями двух разных контекстов:
```
h_mix = (1-α)·h_A + α·h_B
```

Результат — **резко нелинейный**:
```
α=0.00: ст(0.39), стый(0.11), ль(0.05)
α=0.33: \n(0.18), ст(0.11), ка(0.04)
α=0.50: \n(0.74), ((0.01), В(0.01)
α=1.00: \n(0.99), В(0.00), В(0.00)
```

KL(mix || avg. probs) при α=0.5 = **0.805** — переход резкий, не плавный. λ_d работает как **нелинейный фильтр**: даже если состояния смешиваются линейно, LM head переводит смесь в один из режимов, а не в их комбинацию. Это характерно для систем с пороговым поведением.

### 10.4 Gate→h: не регрессия, а routing

Попытка предсказать скрытое состояние h по 4 усреднённым gate-значениям через линейную регрессию:
```
Mean R² = -4.86 (случайный baseline: -1.92)
R² > 0.5: 0/896 dims
```

**Отрицательный R²** означает, что даже простая линейная модель с 4 признаками работает хуже, чем предсказание среднего. Это не регрессия — гейты не кодируют h линейно, они **выбирают режим работы спектрального преобразования**. Одно и то же gate-значение может соответствовать разным h в зависимости от контекста.

**Архитектурный смысл:** гейты — это ручки настройки, а не кодировщики состояния. Их нельзя декодировать обратно в h — это не база, а механизм управления.

## 12. Итоговые выводы по архитектуре

1. **λ_d работает как задумано.** Спектральное внимание (гейты → eigen-моды → блоковое масштабирование) подтверждено эмпирически. Гейты имеют лингвистически осмысленные профили для разных токенов.

2. **V frozen — оптимален.** Случайная ортогональная матрица обеспечивает достаточное перемешивание осей. Учить V не нужно — 9.6M «параметров» можно использовать эффективнее.

3. **K=4 — достаточный минимум.** Четыре спектральных моды (λ=1.618, 1.839, 1.928, 1.966) покрывают необходимый диапазон частот. K=8 быстрее, но разница не драматична.

4. **Гейты — routing, а не кодирование.** 4 gate-значения не могут линейно предсказать h (R² < 0). Они выбирают режим работы, а не кодируют состояние. Это фундаментально отличается от механизма attention в трансформерах.

5. **Интерполяция — нелинейна.** Смесь двух контекстов даёт резкий, а не плавный переход (KL=0.8 при α=0.5). λ_d работает как нелинейный фильтр с пороговым поведением.

6. **Токенизатор оптимален.** Наш 50K BTE, обученный на русской Wikipedia, даёт в 1.7× лучшую компрессию русского текста, чем Qwen3 (146K) или Mistral Nemo (131K).
