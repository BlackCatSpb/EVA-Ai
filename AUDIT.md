# Аудит CogniFlex — Список проблем и задач

**Дата:** 2026-04-06  
**Версия:** 1.0

---

## 1. CRITICAL — Немедленное исправление

### 1.1 /api/status endpoint отсутствует в Web GUI
- **Проблема:** Frontend вызывает `/api/status`, но route зарегистрирован в `eva/server_routes.py` (старый API), а не в `eva/gui/web_gui/server_routes.py`
- **Симптом:** Статус "Ожидание" в боковой панели
- **Решение:** Добавить `/api/status` route в `server_routes.py` (GUI версия)

### 1.2 Navigation buttons работают, но данные не загружаются при первом клике
- **Проблема:** Функции loadLearning, loadKnowledge вызываются только при клике, но данные не загружаются автоматически при первом показе view
- **Решение:** Добавить автозагрузку при переключении view

### 1.3 server_routes_broken.py — дублирующий файл
- **Проблема:** Файл `eva/gui/web_gui/server_routes_broken.py` содержит старый код и не используется
- **Решение:** Удалить файл

## 2. HIGH — Важные исправления

### 2.1 Memory View не загружает данные
- **Функция:** `loadMemory()` в app.js вызывает `/api/metrics`
- **Проблема:** Route существует, но возвращает пустые данные если brain не инициализирован
- **Симптом:** Пустые статистики в Memory View

### 2.2 Analytics View — отсутствует /api/analytics
- **Функция:** `loadAnalytics()` в app.js вызывает `/api/analytics`
- **Проблема:** Route может отсутствовать или возвращать ошибку
- **Решение:** Проверить и добавить endpoint

### 2.3 Knowledge View — некорректная работа
- **Функция:** `loadKnowledge()` вызывает `/api/knowledge-graph`
- **Проблема:** Frontend ожидает данные в определённом формате, backend может не соответствовать

### 2.4 Settings View — проблема с session_id
- **Проблема:** `$('#settingsSession').textContent = activeSessionId || '—';` может показывать undefined
- **Решение:** Проверить инициализацию activeSessionId

### 2.5 loadKnowledgeGraph() вызывается дважды
- **Симптом:** Дублирование запросов при загрузке Memory view
- **Решение:** Оптимизировать логику загрузки

## 3. MEDIUM — Улучшения

### 3.1 Wikipedia -> Поиск — неполное переименование
- **Проблема:** В index.html остались ссылки на "Wikipedia" в sidebar (data-view="wikipedia")
- **Решение:** Переименовать в "Поиск" (уже сделано в HTML, проверить JS)

### 3.2 Brain config — устаревшие параметры wikipedia
- **Параметры:** `categories`, `articles_per_category`, `interval_hours`, `random_per_cycle` больше не используются
- **Решение:** Удалить или пометить как устаревшие

### 33 Отсутствует Health View
- **Симптом:** Код для loadHealth() существует в JS, но view не существует в HTML
- **Решение:** Добавить health view или удалить код

### 3.4 Generation Tracker — избыточный функционал
- **Проблем:** GenerationTracker инициируется, но не используется в полной мере
- **Решение:** Интегрировать с pipeline для отслеживания прогресса

### 3.5 Tavily API — отсутствует fallback при исчерпании квоты
- **Решение:** Добавить fallback на локальный поиск (Wikipedia KB) при исчерпании квоты

## 4. LOW — Оптимизация

### 4.1 server_routes.py — дублирование routes
- **Проблема:** Некоторые routes могут дублироваться между `eva/server_routes.py` и `eva/gui/web_gui/server_routes.py`
- **Решение:** Унифицировать и использовать только GUI версию

### 4.2 EventBusBridge — избыточная логика
- **Проблема:** Bridge патчит старые методы новыми, но не все старые методы используются
- **Решение:** Очистить неиспользуемые patch-и

### 4.3 DeferredCommandSystem — health checks не работают
- **Проблема:** Health checks регистрируются, но не выполняются
- **Решение:** Активировать health monitoring

### 4.4 Pipeline — адаптивные параметры применяются не всегда
- **Решение:** Добавить логирование применения адаптивных параметров

### 4.5 Brain init — избыточные try/except
- **Проблема:** Множественные try/except скрывают реальные ошибки
- **Решение:** Упростить и документировать ошибки
