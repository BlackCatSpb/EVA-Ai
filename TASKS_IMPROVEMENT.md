## P0 — Критичные задачи

### 1. TCM (Temporal Context Memory) — сейчас 40%
**Файл:** `eva_ai/memory/temporal_context.py`, `eva_ai/memory/scenario_tcm.py`

#### Задачи:
- [ ] **TCM-1:** Добавить иерархические временные метки (час, день, неделя, месяц) для сегментов диалога
  - В `TemporalContextMemory` добавить поле `hierarchical_time_label`
  - Реализовать метод `_compute_time_decay(label)` для экспоненциального затухания
  
- [ ] **TCM-2:** Реализовать контрастивное обучение в фоне
  - Создать `ContrastiveTrainer` в `eva_ai/learning/`
  - Обучать на парах (релевантный сегмент, нерелевантный сегмент)
  - Использовать встроенные эмбеддинги для вычисления loss
  
- [ ] **TCM-3:** Улучшить алгоритм извлечения релевантного контекста
  - Учитывать не только семантическую близость, но и временную метку
  - Свежие реплики должны иметь больший вес (exponential decay)
  
- [ ] **TCM-4:** Добавить поддержку типов сегментов (user, assistant, system)
  - Хранить метку типа вместе с эмбеддингом
  - При извлечении учитывать роль говорящего

## Заметки для агентов

1. **При работе с OpenVINO State API** — всегда проверять, что индексы слоев [0-35] соответствуют 36 слоям
2. **При изменении FractalGraphV2** — не забывать обновлять HNSW индекс через `graph_indexer.py`
3. **При доработке LoRA** — использовать `ShadowLoRAManager.atomic_swap()` для бесшовной замены
4. **При изменении KCA/SRG** — тестировать на реальных диалогах с проверкой логических противоречий
5. **При работе с TCM** — учитывать, что эмбеддинги могут быть разных размерностей (512 vs 1024)
