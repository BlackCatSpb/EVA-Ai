## Детали реализации

### 1. Веб-поиск (до LOGIC)

**Требования:**
- Выполняется ДО LOGIC модели
- Оценивает логическую релевантность (используем текущий `_rank_by_relevance`)
- Фильтрует результаты с релевантностью > 0.5
- Форматирует результаты для модели

**Формат вывода:**
```
=== ПОИСК ===
- [Заголовок 1]: релевантность 0.85
  [Краткое содержание]
- [Заголовок 2]: релевантность 0.72
  [Краткое содержание]
```

**Текущий статус:** Уже есть `_rank_by_relevance` в `web_search_integrated.py:970` - работает на основе ключевых слов. Доработка не требуется.

### 2. LOGIC модель (модель A)

**Системный промт:**
```
Дай краткий ответ (1-2 предложения) на запрос.
Затем сформулируй вывод в формате:
=== ВЫВОД === На вопрос о [краткий запрос пользователя] можно сказать что [краткий ответ]
```

**Вход:**
- Запрос пользователя
- Контекст из истории (опционально)
- Результаты веб-поиска

**Выход:**
```
Искусственный интеллект - это технология создания машин, способных обучаться.
=== ВЫВОД === На вопрос о технологии создания машин можно сказать что это ИИ
```

**Извлечение вывода:**
```python
def _parse_logic_output(output: str) -> Tuple[str, str]:
    """Извлекает ответ и вывод из выхода LOGIC."""
    if "=== ВЫВОД ===" in output:
        parts = output.split("=== ВЫВОД ===")
        answer = parts[0].strip()
        conclusion = parts[1].strip()
        return answer, conclusion
    return output, ""
```

### 3. CONTEXT модель (модель B)

**Концепты (формат "Допустим"):**
```
=== КОНЦЕПТЫ ===
Допустим, машинное обучение является основой современного ИИ
Допустим, нейронные сети имитируют работу человеческого мозга
```

**Противоречия (формат "И тем не менее"):**
```
=== ПРОТИВОРЕЧИЯ ===
И тем не менее, ИИ может заменить человеческие рабочие места
И тем не менее, ИИ не обладает сознанием как человек
```

**Поиск концептов/противоречий:**
- Ищем ПО ВЫВОДУ LOGIC (не по изначальному запросу)
- Порог близости: similarity > 0.6 для концептов, > 0.5 для противоречий
- Используем `semantic_search` из FractalGraph

**Системный промт:**
```
Расширь краткий ответ с учетом концепций, рассуждай с учетом противоречий.
Проверь, что расширенный ответ соответствует выводу LOGIC.
Сформулируй вывод в формате:
=== ВЫВОД === На вопрос о [краткий запрос] можно сказать что [расширенный ответ]
```

### 4. Кеширование цепочки обучения

**Формат сохранения (натуральный текст):**
```
Запрос: Что такое искусственный интеллект?
Вывод_A: На вопрос о технологии создания машин можно сказать что это ИИ
Вывод_B: На вопрос о технологии создания машин можно сказать что это область computer science
Ответ: Искусственный интеллект (ИИ) - это...
```

**Ключ кеша:** `learning_chain:{hash(query)}`

**Ограничение:** Максимум 1000 цепочек (удаляем старые при превышении)

### 5. Самодиалог (самообучение)

**Триггер:** `system.idle` событие

**Процесс:**
1. При простое системы (нет запросов > 60 сек)
2. Получить цепочки из HybridCache
3. Проанализировать: запрос → вывод_A → вывод_B → ответ
4. Извлечь знания, концепты, противоречия
5. Записать в FractalGraph

**Интеграция с существующими модулями:**
- `concept_miner.py` - для концептов
- `contradiction_miner.py` - для противоречий
- `dialog_core.py` - для самодиалога

## Формат маркера вывода

**Выбранный маркер:** `=== ВЫВОД ===`

**Причина:** Двойной `===` - редкий символ в тексте, легко парсить
