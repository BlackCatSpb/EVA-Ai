# Карта проекта CogniFlex

## Обзор архитектуры

CogniFlex - это модульная система искусственного интеллекта с ядром, управляющим инициализацией и взаимодействием компонентов. Проект использует фрактальное хранение данных, гибридное кэширование и граф памяти для эффективной работы.

## Структура директорий

### Основные модули

- **`/websearch`** - Веб-поиск и анализ
  - `web_search_engine.py` - Основной класс веб-поиска
  - `search_engines.py` - Поддержка различных поисковых систем
  - `search_models.py` - Модели данных для поиска
  - `database_manager.py` - Управление базой данных поиска
  - `cache_manager.py` - Кэширование результатов поиска

- **`/core`** - Ядро системы и основные компоненты
  - `core_brain.py` - Центральный координатор системы
  - `component_initializer.py` - Инициализатор компонентов
  - `event_system.py` - Система событий
  - `deferred_command_system.py` - Система отложенных команд
  - `system_state.py` - Управление состоянием системы
  - `system_optimizer.py` - Оптимизатор системы
  - `hybrid_token_cache/` - Гибридный кэш токенов
  - `cogniflex_cache/` - Кэширование данных
    - `hybrid_cache/` - Гибридное кэширование
    - `token_cache/` - Кэш токенов
    - `models/` - Модели данных
      - `fractal_store/` - Фрактальное хранилище
      - `memory_graph/` - Граф памяти
  - `background_jobs/` - Фоновые задачи

- **`/mlearning`** - Машинное обучение
  - `ml_unit.py` - Основной модуль ML
  - `cogniflex_tokenizer.py` - Токенизатор CogniFlex
  - `model_manager.py` - Менеджер моделей
  - `training/` - Обучение моделей
  - `inference/` - Вывод моделей

- **`/adaptation`** - Адаптация и обучение
  - `adaptation_core.py` - Ядро адаптации
  - `adaptation_manager.py` - Менеджер адаптации
  - `adaptation_profiles.py` - Профили адаптации

- **`/contradiction`** - Обработка противоречий
  - `contradiction_detection.py` - Обнаружение противоречий
  - `contradiction_resolution.py` - Разрешение противоречий
  - `contradiction_manager.py` - Менеджер противоречий

- **`/ethics`** - Этические аспекты
  - `ethics_manager.py` - Менеджер этики
  - `safety_checks.py` - Проверки безопасности

- **`/distributed`** - Распределенные вычисления
  - `distributed_manager.py` - Менеджер распределенных вычислений
  - `node_communication.py` - Коммуникация между узлами

## Ключевые компоненты

### CoreBrain
- Центральный координатор системы
- Управляет жизненным циклом компонентов
- Обрабатывает системные события
- Управляет ресурсами

### ComponentInitializer
- Инициализирует компоненты
- Управляет зависимостями
- Обеспечивает правильный порядок загрузки

### WebSearchEngine
- Поиск информации в интернете
- Поддержка нескольких поисковых систем (Google, Yandex, Bing)
- Кэширование результатов
- Анализ и фильтрация результатов
- Интеграция с когнитивными возможностями системы

### CogniFlexTokenizer
- Основной токенизатор системы
- Поддержка различных моделей
- Кэширование токенов

### FractalStore
- Фрактальное хранилище данных
- Эффективное хранение и извлечение
- Масштабируемость

### MemoryGraph
- Графовая база знаний
- Хранение ассоциаций
- Семантический поиск

## Конфигурация

Основные параметры конфигурации:
- `device`: Устройство для вычислений (cpu/cuda)
- `cache_dir`: Директория для кэша
- `models_path`: Путь к моделям
- `log_level`: Уровень логирования

## Логирование

Используется модуль `logging` с настройкой уровней:
- ERROR: Критические ошибки
- WARNING: Предупреждения
- INFO: Основная информация
- DEBUG: Отладочная информация
